# TS-021: Add research pipeline variant for non-code tickets

## Objective
Define an alternative pipeline for tickets of type `research` that produces a research document and validation criteria instead of code, implementation plans, and test plans.

## Context
The ticket system supports a `research` type in the frontmatter, but the pipeline always expects `implementation-plan.md` and `test-plan.md`. For research tickets ("Evaluate migration to GraphQL", "Compare state management libraries"), writing a test plan is meaningless — the deliverable is a findings document, not code. This forces users to create fake test plans or skip verification entirely.

## Acceptance Criteria
- [ ] When a ticket has `type: research`, the plan phase generates `research-plan.md` instead of `implementation-plan.md`
- [ ] When a ticket has `type: research`, the plan phase generates `validation-criteria.md` instead of `test-plan.md`
- [ ] `research-plan.md` defines: research questions, sources to investigate, structure of the findings document, decision framework
- [ ] `validation-criteria.md` defines: completeness criteria (all questions answered), evidence requirements (sources cited), deliverable format
- [ ] The implement phase for research tickets produces a `findings.md` document instead of code changes
- [ ] The verify phase for research tickets checks the findings document against validation criteria
- [ ] The conventions skill documents the research pipeline variant

## Technical Approach
Update specs.md:
- Section 3.8: Add `research-plan.md` and `validation-criteria.md` artifact formats
- Section 4.2 `/ticket-system-plan`: Add conditional logic: "If ticket type is `research`, generate research-plan.md and validation-criteria.md instead of implementation-plan.md and test-plan.md"
- Section 4.2 `/ticket-system-implement`: Add conditional: "If ticket type is `research`, follow research-plan.md to produce findings.md"
- Section 4.2 `/ticket-system-verify`: Add conditional: "If ticket type is `research`, verify findings.md against validation-criteria.md"

Research plan format:
```markdown
# Research Plan — PREFIX-XXX

## Questions to Answer
1. Question 1
2. Question 2

## Sources to Investigate
- Source category 1
- Source category 2

## Findings Document Structure
- Section 1: ...
- Section 2: ...

## Decision Framework
How the findings should inform a decision.
```
