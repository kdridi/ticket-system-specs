# TS-005: Add functional test harness for slash command end-to-end testing

## Objective
Provide a repeatable, scripted test environment that exercises the full outer feedback loop (generate → install → init → run commands) so regressions can be caught without manual testing.

## Context
The current outer feedback loop is entirely manual: a developer must run `install.sh`, then `init-project.sh` in a scratch repo, then manually invoke each slash command and observe behavior. This takes significant time per iteration and is inconsistently executed. There is no way to know if a spec change introduced a regression in an untouched command.

## Acceptance Criteria
- [ ] A `test/` directory exists with a `run-tests.sh` script
- [ ] `run-tests.sh` sets up a temporary git repo, installs the generated system, initializes a project, and runs a scripted sequence of slash commands
- [ ] Each test step has a documented expected outcome and emits PASS/FAIL
- [ ] The script cleans up the temporary repo after completion
- [ ] Coverage includes: ticket create, schedule, activate, implement (stub), verify, complete

## Technical Approach
- Use `mktemp -d` to create isolated test repos
- Mock or stub the Claude Code invocation layer where direct execution is not possible
- For steps that require actual Claude Code execution, document them as manual verification steps with expected outputs
- Trap EXIT to ensure cleanup runs even on failure
