# TS-018: Add optional ticket-id argument to /ticket-system-analyze

## Objective
Allow `/ticket-system-analyze` to target a specific ticket by ID, in addition to its default behavior of always analyzing the first ticket on the roadmap.

## Context
Decision D-7 states that analyze always targets the first roadmap ticket, eliminating manual selection for the happy path. However, when the first ticket is blocked by an external dependency, the developer cannot analyze a later ticket directly.

**REJECTED (2026-04-04):** Obsoleted by TS-024 which merges schedule+analyze+split into a unified `/ticket-system-schedule`. The new schedule accepts one or more ticket IDs as arguments. The analyze command no longer exists.
