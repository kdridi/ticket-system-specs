# TS-007: Establish a conventions skill size budget and split strategy

## Objective
Define a clear growth strategy for `ticket-system-conventions` to prevent it from exceeding its 500-line hard limit as new conventions are added, and document how to split it if the limit is reached.

## Context
The `ticket-system-conventions` skill is loaded by every agent and must remain under 500 lines. Currently it covers 10 sections. As the system evolves, new sections (e.g., worktree conventions, artifact schemas, error handling) will push it toward the limit. Without a plan, the skill will silently grow until it breaks the constraint, forcing an unplanned refactor.

The constraint exists because very large skills degrade Claude Code's context efficiency. The limit is architectural, not arbitrary.

## Acceptance Criteria
- [ ] `specs.md` section 6 (Decisions) documents the 500-line budget and how it should be managed
- [ ] A split strategy is defined: which sections can be extracted into a separate `ticket-system-conventions-extended` skill if needed
- [ ] The generated `ticket-system-conventions` skill tracks its current line count in a comment at the top
- [ ] `validate.sh` (TS-002) checks that the skill does not exceed 500 lines

## Technical Approach
- Add a decision entry in `specs.md` section 6 describing the budget and split strategy
- Identify "heavy" sections (plan artifacts, test plan format) as candidates for extraction
- Add a generation rule: emit a `<!-- Lines: N/500 -->` comment at the top of the conventions skill
- Add a `validate.sh` check: `wc -l` on the conventions skill file
